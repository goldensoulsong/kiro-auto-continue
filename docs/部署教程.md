# 详细部署教程

> 本教程面向完全新手。如果你已经熟悉 kiro-rs，直接看根目录 `README.md` 即可。

## 一、准备工作

### 1.1 安装 Node.js

打开 [https://nodejs.org/](https://nodejs.org/)，下载 **LTS 版本**（左边那个绿色按钮），一路下一步装好。

装完打开 `cmd`，输入 `node -v`，能看到类似 `v20.11.0` 的版本号就是装好了。

### 1.2 拿到 Kiro 凭据（refreshToken）

#### 方式 A：已经在用 Kiro IDE

在文件资源管理器地址栏粘贴：

```
%USERPROFILE%\.aws\sso\cache
```

回车后会进入一个文件夹，里面有几个 JSON 文件。挨个用记事本打开，找到带 `refreshToken` 字段的那个，比如 `kiro-auth-token.json`。

文件内容大概长这样：

```json
{
  "accessToken": "aoa...",
  "refreshToken": "aor...",
  "expiresAt": "2026-12-31T...",
  "authMethod": "social",
  "provider": "Google"
}
```

记住 `refreshToken`（aor 开头那一长串）和 `authMethod`（一般是 `social`）。

#### 方式 B：还没注册

去 [https://kiro.dev](https://kiro.dev) 用 Google / GitHub 注册，下载 IDE，登录一次，然后按方式 A 找凭据。

## 二、部署

### 2.1 下载本仓库

打开 cmd，找一个想放的目录（比如 `D:\`）：

```bat
cd /d D:\
git clone https://github.com/goldensoulsong/GitResp2.git
cd GitResp2
```

没装 git 就在浏览器打开 [https://github.com/goldensoulsong/GitResp2](https://github.com/goldensoulsong/GitResp2)，绿色 `Code` 按钮 → `Download ZIP`，解压到 `D:\GitResp2`。

### 2.2 创建凭据文件

仓库根目录里复制一份 `credentials.example.social.json`，**重命名为 `credentials.json`**（去掉 `.example.social`）。

用记事本打开，把 `refreshToken` 替换成你刚才拿到的那串：

```json
[
  {
    "refreshToken": "aor开头的那一长串字符...",
    "expiresAt": "2099-12-31T00:00:00.000Z",
    "authMethod": "social"
  }
]
```

> `expiresAt` 写一个未来的时间就行，程序会自动刷新。

保存。

### 2.3 第一次启动

双击 **`start.bat`**。

如果是第一次运行，会发现 `config.json` 不存在，脚本会自动从模板复制一份并用记事本打开。把里面两个 key 改成你自己的随机字符串：

```json
{
  "apiKey": "sk-kiro-rs-我自己的随机字符串12345abcde",
  "adminApiKey": "sk-admin-我自己的随机字符串67890fghij",
  ...
}
```

> 随机字符串怎么生成？打开 cmd 敲 `node -e "console.log('sk-kiro-rs-' + require('crypto').randomBytes(16).toString('hex'))"` 回车就有了。

保存关闭记事本。**再次双击 `start.bat`**。

### 2.4 验证启动成功

cmd 窗口看到：

```
============================================================
  kiro-rs Auto-Continue 一键启动
============================================================
  kiro-rs 后端:  http://127.0.0.1:8991
  代理端口:      http://127.0.0.1:8992  ← 酒馆连这个
  管理面板:      http://127.0.0.1:8993
============================================================
```

5 秒后浏览器自动打开 [http://127.0.0.1:8993](http://127.0.0.1:8993)。

切到 "凭据管理（kiro-rs）" 标签，能看到你的账号和余额，说明 OK 了。

## 三、接入客户端

### 3.1 SillyTavern（酒馆）

设置 → API：

- **API**：`Claude`
- **Claude API Key**：你刚才设的 `apiKey`（sk-kiro-rs-... 那个）
- **Custom Endpoint**：勾上，填 `http://127.0.0.1:8992`
- **Model**：`claude-sonnet-4-20250514` 或 `claude-opus-4-20250514`

点 Connect，绿灯就好了。

### 3.2 Cline / Continue（VSCode 插件）

API Provider：`Anthropic`
API Key：同上 `apiKey`
Custom Base URL：勾上填 `http://127.0.0.1:8992`

## 四、日常使用

### 4.1 启动 / 关闭

- 启动：双击 `start.bat`
- 关闭：直接 X 掉 cmd 窗口，所有服务（kiro-rs + 自动续写代理 + 管理面板）一起停

### 4.2 调整自动续写

打开管理面板 [http://127.0.0.1:8993](http://127.0.0.1:8993)：

- **总开关**：关掉就只透传，不续写
- **字数触发阈值**：默认 7600。Claude 输出超过这个字数且没正常结束就触发续写。如果觉得续写不够积极，调小（如 6000）；如果觉得乱续写，调大
- **最大续写次数**：单条消息最多续写几次。默认 5 次（即一条消息最多输出 5×8000 ≈ 40000 token），可以调到 10
- **限流**：默认 60 秒 3 次。SillyTavern 群聊场景容易超限，可以调大窗口内次数到 10

改完点切换/输入框失焦就生效，不用重启。

### 4.3 凭据管理

切到管理面板的 "凭据管理（kiro-rs）" 标签：

- 看余额 / 用量
- 添加新凭据
- 删除 / 禁用 / 调优先级

## 五、排错

### 5.1 启动时 cmd 闪退

cmd 改用"以管理员身份运行"，再 `cd` 到目录手动跑 `start.bat`，看错误信息。

### 5.2 浏览器打开管理面板是 ERR_CONNECTION_REFUSED

cmd 还没跑起来。等 10 秒再刷新。如果一直不行，看 cmd 窗口报错。

### 5.3 cmd 报 `[互斥保护] 检测到 Kiro IDE 正在运行`

任务管理器关掉 `Kiro.exe`，再启动。或者把 `tools\auto-continue\config.json` 里 `ideExclusiveEnabled` 改成 `false`（**不推荐**，同号并发可能被封）。

### 5.4 客户端连接报 401 / Unauthorized

客户端填的 API Key 和 `config.json` 里的 `apiKey` 没对上。复制粘贴时注意没有多余空格。

### 5.5 客户端连不通 / 一直 Network Error

- 端口对不对？是 **8992** 不是 8991
- 防火墙挡了？Windows Defender 第一次启动可能弹窗，点"允许访问"
- VPN / 代理软件干扰？把 127.0.0.1 加到代理白名单

### 5.6 输出还是被截断没续写

管理面板看 "总续写" 数字。如果一直是 0：
- 总开关确实是开的吗？
- `truncateThreshold` 是不是设太大了？调到 5000 试试

### 5.7 Token 刷新失败 / error request

`config.json` 里 `tlsBackend` 从 `rustls` 改成 `native-tls`，重启。

## 六、卸载

直接删掉整个目录就行。没装系统服务、没改注册表。

如果还想清理 `kiro-rs.exe` 留下的别处缓存：没有，本工具所有数据都在自己目录里。
