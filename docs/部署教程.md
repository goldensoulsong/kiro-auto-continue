# 详细部署教程

> 本教程面向完全新手。如果你已经熟悉 kiro-rs，直接看根目录 `README.md` 即可。

## 一、准备工作

### 1.1 安装 Node.js

打开 [https://nodejs.org/](https://nodejs.org/)，下载 **LTS 版本**（左边那个绿色按钮），一路下一步装好。

装完打开 `cmd`，输入 `node -v`，能看到类似 `v20.11.0` 的版本号就是装好了。

### 1.2 （可选）了解一下 Kiro 凭据是什么

简单说就是登录 Kiro IDE 后本地存的一份 token，反代靠它代你向 AWS 发请求。如果你电脑里登录过 Kiro IDE，这份 token 在：

```
%USERPROFILE%\.aws\sso\cache\
```

里面会有几个 JSON 文件，其中至少一个含 `refreshToken` 字段。**这一步不用你手动找了**，后面 start.bat 起来后管理面板能一键扫描导入。

如果还没登录过 Kiro IDE，去 [https://kiro.dev](https://kiro.dev) 用 Google / GitHub 注册，下载 IDE，登录一次，关掉 IDE。

## 二、部署

### 2.1 下载本仓库

打开 cmd，找一个想放的目录（比如 `D:\`）：

```bat
cd /d D:\
git clone https://github.com/goldensoulsong/kiro-auto-continue.git
cd kiro-auto-continue
```

没装 git 就在浏览器打开 [https://github.com/goldensoulsong/kiro-auto-continue](https://github.com/goldensoulsong/kiro-auto-continue)，绿色 `Code` 按钮 → `Download ZIP`，解压到 `D:\kiro-auto-continue`。

### 2.2 启动

双击 **`start.bat`**。

第一次启动会自动做这些事，你**啥都不用填**：

```
[提示] config.json 不存在，正在自动生成...
       已生成 config.json 并自动填入随机 apiKey / adminApiKey

[提示] credentials.json 不存在，正在生成空文件
       启动后请打开管理面板（http://127.0.0.1:8993）使用"快捷导入"添加凭据

------------------------------------------------------------
客户端（酒馆/Cline等）接入信息：
  API URL: http://127.0.0.1:8992
  API Key: sk-kiro-rs-xxxxxxxxxxxxxxxxxxxxxxxx...
------------------------------------------------------------

启动中... 5 秒后将自动打开管理面板
关闭本窗口可同时停止所有服务
```

**API Key** 那一行你后面要复制到客户端里，先记住位置。

5 秒后浏览器自动打开 [http://127.0.0.1:8993](http://127.0.0.1:8993)。

> 第一次浏览器没自动打开？手动打开 [http://127.0.0.1:8993](http://127.0.0.1:8993) 即可。

### 2.3 导入 Kiro 凭据

进管理面板默认就在 **快捷导入** 标签页，三种方式任选：

#### ① 扫描本机凭据（推荐）

如果你电脑上登录过 Kiro IDE，点 **扫描本机凭据**，会扫出来类似：

```
找到 1 个候选凭据，勾选要导入的：
☑ refreshToken: aor-12345abc...
   authMethod: social
   expiresAt: 2026-...
   C:\Users\xxx\.aws\sso\cache\xxx.json
```

直接点 **导入选中** 完成。

#### ② 粘贴 token JSON

朋友把 `kiro-auth-token.json` 文件全文给你了，复制到文本框：

```json
{"accessToken":"...","refreshToken":"aor...","expiresAt":"2026-...","authMethod":"social"}
```

点 **解析并导入**。

#### ③ 粘贴 refreshToken

只有 `aor...` 那一长串，把它粘到 refreshToken 输入框，认证方式选 `social`（个人账号）或 `idc`（企业账号），点 **导入**。IdC 还要填 `clientId` 和 `clientSecret`。

### 2.4 验证导入成功

切到 **凭据管理（kiro-rs）** 标签，能看到刚导入的凭据出现在列表里，含余额、订阅类型、过期时间等信息。

如果列表空着或一直转圈，等 5~10 秒再刷新，kiro-rs 后端首次拉数据稍慢。

## 三、接入客户端

### 3.1 SillyTavern（酒馆）

设置 → API：

- **API**：`Claude`
- **Claude API Key**：`start.bat` 窗口里打印的 `sk-kiro-rs-...`
- **Custom Endpoint**：勾上，填 `http://127.0.0.1:8992`
- **Model**：`claude-sonnet-4-20250514` 或 `claude-opus-4-20250514`

点 Connect，绿灯就好了。

### 3.2 Cline / Continue（VSCode 插件）

- API Provider：`Anthropic`
- API Key：同上
- Custom Base URL：勾上填 `http://127.0.0.1:8992`

> 后续要查看 API Key，打开仓库目录的 `config.json` 找 `apiKey` 字段。

## 四、日常使用

### 4.1 启动 / 关闭

- 启动：双击 `start.bat`
- 关闭：直接 X 掉 cmd 窗口，所有服务（kiro-rs + 自动续写代理 + 管理面板）一起停

### 4.2 调整自动续写

打开管理面板 [http://127.0.0.1:8993](http://127.0.0.1:8993)，切到 **自动续写 / 服务** 标签：

- **总开关**：关掉就只透传，不续写
- **字数触发阈值**：默认 7600。Claude 输出超过这个字数且没正常结束就触发续写。觉得续写不够积极调小（如 6000），觉得乱续写调大
- **最大续写次数**：单条消息最多续写几次。默认 5 次（一条消息最多输出 5×8000 ≈ 40000 token），可以调到 10
- **限流**：默认 60 秒 3 次。SillyTavern 群聊场景容易超限，可以调大窗口内次数到 10

改完点切换/输入框失焦就生效，不用重启。

### 4.3 凭据管理

切到 **凭据管理（kiro-rs）** 标签：

- 看余额 / 用量
- 添加新凭据（也可以回快捷导入用扫描方式）
- 删除 / 禁用 / 调优先级

### 4.4 多账号轮换

通过快捷导入多次添加凭据即可，每条凭据自动分配 ID。在 **凭据管理（kiro-rs）** 里调整每条的 `priority`：

- 数字越小优先级越高（0 最优先）
- 当前活跃凭据失败时自动切到下一条

## 五、排错

### 5.1 启动时 cmd 闪退

cmd 改用"以管理员身份运行"，再 `cd` 到目录手动跑 `start.bat`，看错误信息。最常见：没装 Node.js、端口冲突、当前目录路径有特殊字符。

### 5.2 浏览器打开管理面板是 ERR_CONNECTION_REFUSED

cmd 还没跑起来。等 10 秒再刷新。如果一直不行，看 cmd 窗口报错。

### 5.3 cmd 报 `[互斥保护] 检测到 Kiro IDE 正在运行`

任务管理器关掉 `Kiro.exe`，再启动。或者把 `tools\auto-continue\config.json` 里 `ideExclusiveEnabled` 改成 `false`（**不推荐**，同号并发可能被封）。

### 5.4 客户端连接报 401 / Unauthorized

客户端填的 API Key 和 `config.json` 里的 `apiKey` 没对上。复制粘贴时注意没有多余空格。要看当前的 apiKey，直接打开 `config.json` 找 `apiKey` 字段。

### 5.5 客户端连不通 / 一直 Network Error

- 端口对不对？是 **8992** 不是 8991
- 防火墙挡了？Windows Defender 第一次启动可能弹窗，点"允许访问"
- VPN / 代理软件干扰？把 127.0.0.1 加到代理白名单

### 5.6 输出还是被截断没续写

管理面板看 "总续写" 数字。如果一直是 0：
- 总开关确实是开的吗？
- `truncateThreshold` 是不是设太大了？调到 5000 试试

### 5.7 Token 刷新失败 / error request

`config.json` 里 `tlsBackend` 从 `rustls` 改成 `native-tls`，重启。

### 5.8 调用报 `400 缺少 profile arn`

通常是 IdC 账号没正确填 `clientId` / `clientSecret`，或者凭据 region 配错。

- Social 账号一般不会遇到这个问题
- 用快捷导入的扫描方式比手动填出错率低很多
- 如果还有问题，先在 **凭据管理（kiro-rs）** 里删除那条凭据，重新导入

### 5.9 扫描本机凭据扫不到东西

1. 确认电脑上**登录过** Kiro IDE（不是只装了没登录）
2. 在文件资源管理器地址栏粘贴 `%USERPROFILE%\.aws\sso\cache` 看里面有没有 JSON 文件
3. 实在扫不到就改用粘贴方式（方式 2 / 3）

## 六、卸载

直接删掉整个目录就行。没装系统服务、没改注册表，本工具所有数据都在自己目录里。
